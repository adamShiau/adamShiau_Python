Aegiverse AHRS / GNSS Script

Author: Adam Shiau
Date: 2025-12-18
Script: aegiverse_g_ahrs_v2.py
Current Version: 2.0

Requirement

pip install pyserial

Basic Usage

a. List available COM ports

python aegiverse_g_ahrs_v2.py list

b. Start reading data

Note: After powering on the device, please wait about 5 seconds before starting data reading.

b.1 AHRS packet (mode = 1)

Windows:

python aegiverse_g_ahrs_v2.py 1 COM27

Linux:

python aegiverse_g_ahrs_v2.py 1 /dev/ttyUSB0

b.2 GNSS packet (mode = 2)

Windows:

python aegiverse_g_ahrs_v2.py 2 COM27

Linux:

python aegiverse_g_ahrs_v2.py 2 /dev/ttyUSB0

c. Stop reading data

c.1 Send stop command

Windows:

python aegiverse_g_ahrs_v2.py 0 COM27

Linux:

python aegiverse_g_ahrs_v2.py 0 /dev/ttyUSB0

c.2 Keyboard interrupt

Press Ctrl + C

Output Format

AHRS output (mode = 1)

TIME(s),WX(°/s),WY(°/s),WZ(°/s),AX(m/s²),AY(m/s²),AZ(m/s²),PD_TEMP(°C),PITCH(°),ROLL(°),YAW(°)

GNSS output (mode = 2)

MCU_TIME(s),LATITUDE(deg),LONGITUDE(deg),ALTITUDE(m),
UTC_YEAR,UTC_MONTH,UTC_DAY,
UTC_HOUR,UTC_MINUTE,UTC_SECOND,UTC_MILLISECOND,
GPS_STATUS_CODE

Formatting rules:

  • MCU_TIME: 3 decimal places (seconds)
  • Latitude / Longitude: 6 decimal places (degrees)
  • Altitude: 2 decimal places (meters)

Version History / Change Log

v2.0 (2025-12-18)

  • Added GNSS packet support (mode=2).

v1.0 (2025-12-01)

Notes

  • Mode 1 and mode 2 are mutually exclusive; only one packet type is streamed at a time.
  • GNSS MCU_TIME is derived from the device MCU timer.
  • GPS_STATUS_CODE definition follows the device protocol.
